## **DUAL EXFILTRATION STRATEGY ANALYSIS**

### **Why Two Separate Waves?**

The attacker used a sophisticated dual-wave strategy:

**Wave 1 (11:20-11:22):** General Corporate Data
- Board minutes, financial reports, strategic plans
- Lower risk documents
- Test exfiltration infrastructure
- 156 MB transferred successfully

**5-Minute Operational Pause**
- Verify Wave 1 success
- Prepare for high-value targets
- Stage most sensitive documents

**Wave 2 (11:27-11:29):** Crown Jewels
- M&A acquisition details (insider trading risk)
- Patent applications (IP theft)
- Network architecture (future attack planning)
- Active Directory documentation (persistence planning)
- **Encrypted immediately before exfiltration** (dual extortion)
- 89 MB of the most damaging data

### **Tactical Advantages:**

1. **Risk Distribution** - If Wave 1 detected, Wave 2 could be aborted
2. **Priority Targeting** - Most valuable data saved for last
3. **Dual Extortion Setup** - Encrypt high-value files before stealing
4. **Verification Time** - Confirm Wave 1 upload before continuing
5. **Operational Security** - Smaller, faster transfers less likely to trigger alerts

2. **Data Loss Prevention (DLP)**
   - Inspect all HTTP/HTTPS POST requests
   - Block uploads >50 MB to external IPs
   - Alert on archive file (.zip) uploads
   - Categorize and block sensitive document exfiltration

3. **Network Egress Controls**
   - Allow outbound HTTP only to approved destinations
   - Block all traffic to 185.220.101.0/24
   - Alert on sustained outbound transfers >10 MB
   - Implement DNS filtering

4. **PowerShell Monitoring**
   - Block PowerShell execution from Office applications
   - Enable Constrained Language Mode
   - Log all Compress-Archive and Invoke-WebRequest commands
   - Alert on PowerShell network connections

   - Correlate: archive creation + network upload + file deletion
   - Alert on Domain Admin accounts performing file operations
